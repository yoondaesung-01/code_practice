# [LeetCode] 1378. Replace Employee ID With The Unique Identifier

## 1. ❓ 문제 정의
Write a solution to show the unique ID of each user, 
If a user does not have a unique ID replace just show null.
Return the result table in any order.

- **목표:** 각 사용자의 `unique_id`와 `name`을 함께 보여주어야 한다.
- **조건:**
    1. 만약 사용자에게 `unique_id`가 없다면, `null`로 표시해야 한다.
    2. 데이터가 없는 사용자(Alice, Bob 등)도 결과에서 사라지면 안 된다.

## 2. 💡 풀이 접근
- 두 테이블(`Employees`, `EmployeeUNI`)을 합쳐야 하므로 `JOIN`을 사용한다.
- **핵심 조건:** `unique_id`가 없는 직원도 출력되어야 한다.
    - `INNER JOIN`을 쓰면 짝이 없는 데이터는 삭제되므로 사용할 수 없다.
    - 왼쪽 테이블(`Employees`)의 데이터를 모두 살리기 위해 **`LEFT JOIN`**을 사용해야 한다.
- 두 테이블을 연결하는 기준(`ON`)은 `id` 컬럼이다.

## 3. 📝 정답 코드 (PostgreSQL)
```PostgreSQL
SELECT unique_id, name
FROM Employees 
LEFT JOIN EmployeeUNI 
ON Employees.id = EmployeeUNI.id;
```
### ✅ 체크 포인트
- LEFT JOIN 문법:
```PostgreSQL
SELECT 가져올_컬럼들
FROM 기준이_되는_테이블 (모두 보여줘야 하는 테이블)
LEFT JOIN 덧붙일_테이블 (정보가 없으면 NULL이 될 테이블)
    ON 두_테이블의_연결고리 (공통 컬럼 (ex: Employees.id = EmployeeUNI.id))
```
- **기억 안나는 JOIN 문제 주의깊게 보기!**
